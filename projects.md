## Development Approach

I build tools with a **planning-first** approach. Every project lives in a `.planning/` directory, where I define core values in `PROJECT.md`, track execution in `STATE.md`, and verify progress through rigorous phase-based goals.

### Core Principles

- **Solve real problems**: Tools address specific needs, not hypothetical use cases
- **Type safety**: Catch errors early through strong typing
- **Test thoroughly**: Comprehensive tests validate correctness
- **Document decisions**: Capture rationale for future reference
- **Plan systematically**: Phase-based execution with verification
- **Keep it simple**: Avoid over-engineering and unnecessary abstraction
- **Make it fast**: Performance matters for large-scale network topologies
- **Build modularly**: Clean interfaces enable reuse and evolution
